## ESTRUCTURA DE DATOS PROPUESTA

### Areas.js
```
'sql-base'           → Curso Base SQL
'postgresql'         → PostgreSQL
'sql-server'         → SQL Server
'mysql'              → MySQL
'oracle'             → Oracle
```

### Niveles.js
- Mantener nivel1-nivel8 con areaId dinámico
- Cada área tendrá sus propios niveles

### Ejercicios.js
- 70 ejercicios base con areaId: 'sql-base'
- 70 ejercicios base con areaId: 'postgresql' (copiados + adaptados)
- 70 ejercicios base con areaId: 'sql-server' (copiados + adaptados)
- 70 ejercicios base con areaId: 'mysql' (copiados + adaptados)
- 70 ejercicios base con areaId: 'oracle' (copiados + adaptados)
- + Ejercicios específicos de cada motor

## VENTAJAS DE ESTA REORGANIZACIÓN

✅ El usuario puede empezar con SQL Estándar (base sólida)
✅ Luego especializarse en su motor de BD preferido
✅ Reutilización de ejercicios base (DRY principle)
✅ Conceptos comunes en todos los motores
✅ Características específicas claramente separadas
✅ Progresión clara de dificultad en cada especialidad
✅ El progreso se guarda por areaId (usuario puede estar en nivel3 de SQL Base y nivel1 de PostgreSQL)
